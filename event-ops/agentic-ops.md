# Agentic Event Operations — Menu

> v0.1 — July 2026. The event about agents should be *run* by agents where it helps. This is an à-la-carte menu, not a commitment — each item stands alone, with realistic build effort (hours, not days, using current agentic coding tools) and what's needed from organizers to build it.

| # | Capability | What it does | Inputs needed | Build effort |
|---|---|---|---|---|
| 1 | **Registration & team formation** | Form → roster tracker; agent proposes balanced teams (section mix, self-reported comfort with AI tools) and drafts confirmation emails for organizer review | Registration form fields; team-size rule | ~3–4 hrs |
| 2 | **Comms drafting** | One source-of-truth event doc → agent drafts every announcement, reminder, and FAQ reply on schedule; humans review and send everything | Event doc; comms calendar; channel list | ~2–3 hrs |
| 3 | **Submission intake & validation** | Checks each submission for completeness (deck, demo video, process log), flags gaps to the team before the deadline instead of after | Submission form; the gate checklist from [../case/spec.md](../case/spec.md) | ~2–3 hrs |
| 4 | **Judge ops** | Generates per-judge scoring sheets from [../case/rubric.md](../case/rubric.md), aggregates scores, applies weights and tiebreak, drafts feedback summaries back to every team (not just winners) | Final rubric; judge list | ~3–4 hrs |
| 5 | **Event site** | Public one-pager on GitHub Pages from this repo: what/when/register, schedule, FAQ, sponsor placement | Confirmed date + registration link | ~2 hrs — **deferred until the event is confirmed** |

## Principles

- **Humans send everything.** Agents draft; organizers approve. No student or sponsor ever receives an unreviewed machine message.
- **One source of truth.** Every agent reads from the same event doc, so a date change propagates everywhere without a scavenger hunt.
- **The ops stack is itself a demo.** Running the event this way is a quiet proof of the competition's whole thesis — worth a slide in the kickoff deck.

## Sequencing

Nothing here blocks anything else. Natural order once the event is real: site (#5) and registration (#1) first, comms (#2) alongside, intake (#3) and judge ops (#4) built during the registration window. Total stack: roughly one focused weekend.
